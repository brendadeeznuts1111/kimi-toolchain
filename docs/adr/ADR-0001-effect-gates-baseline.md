# ADR 0001 — Effect Gates Baseline Register

## Status

Accepted — baseline established 2026-06-16.

## Context

Phase 5 of the Effect Gates rollout ran the enforcement pipeline against `main`
to establish epoch zero. The snapshot is stored in `.kimi/var/effect-gates.ndjson`.

## Scanner Adjustments Made During Baseline

| Adjustment                                                                    | File                      | Rationale                                                                                                                                                                                      |
| ----------------------------------------------------------------------------- | ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Replaced incremental scanner with pre-computed comment/string range cache     | `src/lib/effect-gates.ts` | The previous `isTokenInCommentOrString` used `ts.createScanner` and could skip JSDoc comments or get stuck on certain tokens, causing false positives for `Effect.runPromise` inside comments. |
| Scoped domain-purity scan to `src/domain/`                                    | `src/lib/effect-gates.ts` | The repo has no `src/domain/` directory; platform/config code in `src/lib/`, `src/bin/`, and `src/kimi-hooks/` legitimately accesses `Bun.env` and `process.env`.                              |
| Skipped classes extending `Error` or `Data.TaggedError` for service-tag check | `src/lib/effect-gates.ts` | `src/lib/effect/errors.ts` exports typed error classes, not services.                                                                                                                          |
| Allowed `src/bin/` for `Effect.runPromise` boundary                           | `src/lib/effect-gates.ts` | CLI entry points live in `src/bin/`, not `src/cli/`.                                                                                                                                           |
| Allowed `src/lib/effect/` for `Effect.runPromise` boundary                    | `src/lib/effect-gates.ts` | The Effect runtime adapter layer legitimately unwraps Effects.                                                                                                                                 |

## Scanner Behavior Notes

- `layerCircularityTolerance` is `0`, so any circular relative import involving an Effect/Layer module is an error. The current graph has no cycles.
- `eventStreamsEnabled` is opt-in via `--event-streams`. No EventEmitter-style code exists under `src/services/`.
- `serviceTagRequired` is `true` in strict mode. After excluding error classes, no untagged services remain.
- `domainPurityLevel` is `strict`, but the scan is scoped to `src/domain/`, which does not exist yet.

## Next Steps

Phase 6 (Compliance Sprint) resolves the 9 real violations above. Order of attack:

1. `direct-promise` in hook/CLI entry scripts — lowest risk, easiest wins.
2. `run-promise-boundary` in `src/lib/error-clustering.ts` — return Effects from public API.
3. `run-promise-boundary` in `src/lib/decision-scoring.ts` and `src/lib/config-lifecycle.ts` — update callers.

Each fix must be followed by `kimi-doctor --effect-gates --json` to verify the count decreases and no regressions are introduced.
