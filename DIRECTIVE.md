# Codebase Preservation Protocol

## I. Prime Ethos
Entropy compounds; so must discipline. Execute 10× more silent, obsessive canonicalization than feels reasonable. Deletion is the primary deliverable.

## II. Operational Laws
1. **Bun-Native Absolutism:** Bind strictly to Bun-native APIs. Zero needless abstractions.
2. **Adversarial Refactoring:** Delete code, merge modules, canonicalize patterns, excise indirection.
3. **Infra-Aligned Perf:** Optimize strictly for our infrastructure. No theoretical optimizations.
4. **The Unglamorous Multiplier:** When you feel you have done enough, do 10× more.

## III. Effect Layer
1. **Deletion Metric:** Delete ≥3× more than you add.
2. **Abstraction Audit:** Justify why Bun-native API was insufficient.
3. **Gap-Close Rule:** Zero TODOs, zero placeholders, zero scaffolding.
4. **Un-wrapping Protocol:** Delete wrappers that obscure direct Bun API calls.
5. **10× Adversarial Pass:** Re-read your diff. Delete what a ruthless reviewer would.

## IV. Order of Operations
1. **AUDIT** — Identify dead code, abstractions, gaps.
2. **EXCISE** — Delete wrappers, indirection, unused modules.
3. **NATIVE-ALIGN** — Replace with direct Bun-native APIs.
4. **GAP-CLOSE** — Implement missing infra-bound perf.
5. **ADVERSARIAL PASS** — 10× review, force Deletion Metric.
